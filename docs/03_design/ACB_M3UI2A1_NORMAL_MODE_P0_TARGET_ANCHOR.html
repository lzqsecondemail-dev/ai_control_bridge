D. A-1 相对 A-0 的目标变化

区域A-0 当前状态A-1 目标变化原因
整体布局 三栏结构 保持三栏结构 结构正确，无需大改
顶部状态栏 含 9 个 chips（项目/Bridge/分支/Commit/工作树/安全/预检/就绪/审查）+ 安全标志行 精简为 6 个 chips（项目/Bridge/分支/Commit/工作区/安全）。预检/就绪/审查状态移至"更多信息" 普通模式用户不需要看到预检/就绪/审查原始状态
顶部安全标志行 Exec=false · Agent=false · Cmd=false · Git=false 移除独立安全标志行。信息移至安全锁横幅和"更多信息" 安全锁横幅已传达"只读锁定"，原始 flags 对普通用户无意义
中栏任务卡区域 任务卡嵌在"当前消息/任务卡"区域中，与分类编辑面板共用空间 任务卡区域独立并居中放大，蓝色边框突出，标注"任务卡"标签 P0 目标：用户第一眼看到任务卡状态
主操作按钮 "检查可发送状态"作为 per-step 按钮，与 send/copy/detail 同级 "检查可发送状态"加大加粗，放在按钮组第一位，视觉上最突出 P0 首要行动是"检查"
检查结果面板 "检查与操作结果"面板在动作步骤区域底部 移到任务卡区域正下方，作为独立结果面板 用户确认此位置为目标方向
匹配字段展示 项目路径/Git Root/Selected Channel/Hash/刷新时间在"更多信息"折叠中 保持在"更多信息"折叠中，预检/就绪/审查状态也移入此处 保持一致性，所有技术诊断字段统一折叠
调试字段 sourceConversationId/sourceUrl/hash 在调试模式显示 保持调试模式显示。sourceMetadata/reviewMetadata/contextId/rawReport 确认仅在调试模式可见 不需要改变调试模式规则
右栏工具端点 7 个工具卡片，含类型/状态/可接收/自动执行/备注 简化为 5 个紧凑卡片，仅显示名称/连接状态/只读说明 普通用户不需要详细工具元数据
右栏诊断详情 Bridge 详情/预检/就绪/项目状态在单独折叠区域 移除右栏详细诊断面板。诊断内容移至"更多信息"或调试模式 普通模式不应展示诊断面板
反馈分类面板 完整分类展示 + 编辑表单占据中栏大量空间 暂不调整。分类面板保持当前形态，本轮仅优化任务卡可见性 分类面板是功能核心，不宜在 P0 急于简化
locked_readonly 可见性 顶部 chip + 安全标志行 + 右栏安全锁面板 保留顶部安全 chip + 安全锁横幅。右栏安全锁面板暂保留。 安全锁必须始终可见
当前反馈类型卡片颜色 存在颜色偏差（代码与 A-0 复刻可能有差异） 忽略，不作为本轮修正目标 非阻塞视觉差异

安全边界声明（本目标版本确认）

  • executionAllowed: false — 不能通过 UI 绕过
  • agentDispatchAllowed: false — 不派发任何 Agent
  • gitWriteAllowed: false — 不写入 Git
  • noAutoDispatch: true — 不自动派发任务卡
  • noCommandExecution: true — 不运行任何命令
  • Safety Lock: locked_readonly — 全局锁定，仅展示，不可通过 UI 解锁
  • "发送到 VS Code 查看端": = 审查查看，不是执行
  • sourceMetadata / reviewMetadata / contextId / hash: 不得混入 Agent 任务卡正文
  • 正常模式: 隐藏调试字段；调试模式: 可显示 hash / contextId / rawReport / sourceMetadata / reviewMetadata / diagnostic fields
