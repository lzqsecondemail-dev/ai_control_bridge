# ACB UI 基线工作流

> 适用范围：ACB 浏览器控制台、VS Code 查看端及后续所有 UI 可视化变更的开发治理。  
> 当前定位：UI 开发治理工作流规则，不是 UI 代码实现。  
> 创建日期：2026-06-01

---

## 1. 目的

ACB UI 开发已进入需要可视化验收锚点的阶段。纯文字任务卡在 UI 结构变更中不够可靠，导致反复小修和高沟通成本。

本工作流不增加工具链成本，不强制引入 Figma/v0/Uizard，仅在现有基础上增加一个结构化的 UI 基线交接环节。

核心工作流句子：

> **总控定版本，UI 定界面基准，总控派任务，执行端落地，UI 回看，总控收口。**

---

## 2. 角色边界

| 角色 | 职责 | 禁止 |
|------|------|------|
| **执行总控** | 定义版本目标、范围和最终任务卡；拆分实现任务；最终验收收口 | — |
| **UI/UX 顾问** | 产出 UI 基线、状态矩阵、按钮文案表、字段层级、审查建议 | **不得直接派发实现任务** |
| **用户** | 确认方向、提供标注和截图、最终决策 | — |
| **DeepSeek / Codex / Claude Code** | 仅执行总控批准的任务卡 | 不得自行扩大范围、跳过门禁、或根据蓝图直接全面改写 UI |
| **其他专项顾问** | 提供专项建议，回到总控收口 | 不得绕开总控直接派发执行 |

### 2.1 UI/UX 顾问的硬边界

- UI/UX 顾问的建议必须经过总控审查后才可转化为实现任务卡。
- UI/UX 顾问不得直接向 DeepSeek / Codex / Claude Code / PowerShell 发送任务卡。
- UI/UX 顾问不得指示执行端执行命令、修改文件和提交代码。
- 用户始终保留对顾问建议的修改权和否决权。

---

## 3. 版本类型规则

不是所有 UI 变更都需要完整蓝图。根据变更类型采用不同的交接要求。

| 版本类型 | 必需工件 | 不需要 |
|----------|----------|--------|
| **UI 结构版本**（布局重排、新增区域、三栏调整、抽屉等） | 目标 UI 图片 / 静态蓝图页 / before-after 对比图 | 不需要状态矩阵（如果交互不变） |
| **交互状态版本**（按钮启用/禁用、loading/error/empty 状态变化、模式切换） | 状态矩阵表 | 不需要完整蓝图（如果布局不变） |
| **后端/Bridge/门禁版本**（Preflight、Readiness、Bridge 端点变更） | 字段定义、输入/输出示例、测试报告示例 | 不需要 UI 图片 |
| **小修小改**（文案修正、颜色调整、间距微调、单一状态修正） | 局部 before/after 截图或局部状态说明 | 不需要完整蓝图、状态矩阵 |

### 3.1 混合版本

如果一个任务卡同时涉及 UI 结构和交互状态变更（如"新增三栏布局 + 每栏不同状态显示"），则两种工件都需要：

- 目标 UI 蓝图（结构变更）
- 状态矩阵（交互状态变更）

## 4. UI 基线包模板

当 UI/UX 顾问交接 UI 结构版本时，应产出以下包：

### 4.1 目标 UI 图片或静态蓝图页

- 静态 HTML 蓝图页（如 `docs/03_design/acb_console_ui_blueprint.html`）
- 或浏览器全页截图
- 或 before-after 对比图

### 4.2 变更列表

逐项列出本版本的 UI 变更:
- 新增元素
- 删除元素
- 移动元素
- 折叠/展开变更
- 重命名标签
- 信息层级调整

### 4.3 状态/按钮验收表

对每个可交互元素列出所有可能状态:

| 元素 | 状态 | 预期显示 |
|------|------|----------|
| "检查可发送状态" 按钮 | 未运行 | 默认蓝色按钮 |
| "检查可发送状态" 按钮 | 运行中 | 按钮 + loading 指示 |
| "检查可发送状态" 按钮 | 通过 | 绿色结果面板 |
| "发送到 VS Code 查看端" 按钮 | 未检查 | 禁用（灰色） |
| "发送到 VS Code 查看端" 按钮 | 可发送 | 启用（蓝色） |
| ... | ... | ... |

### 4.4 正常/调试模式字段层级

| 层级 | 可见条件 | 包含字段 |
|------|----------|----------|
| Level 1 | 正常模式，始终可见 | 状态栏、任务卡区域、按钮、操作结果、安全锁 |
| Level 2 | 正常模式，折叠/hover/抽屉 | 任务详情、工具端点详细信息 |
| Level 3 | 仅调试模式 | hash、contextId、rawReport、sourceMetadata、reviewMetadata、diagnostic fields |

### 4.5 禁止范围 (Out-of-Scope)

明确声明本版本**不实现**的内容:
- 不开放的命令、端点、权限
- 不覆盖的 UI 区域
- 不处理的状态/边界情况
- 不在本迭代中实现的功能

### 4.6 安全边界声明

每个 UI 基线包必须包含以下声明的确认:

```text
安全边界声明（本 UI 基线版本确认）:
- executionAllowed: false（不能通过 UI 绕过）
- agentDispatchAllowed: false
- gitWriteAllowed: false
- noAutoDispatch: true
- noCommandExecution: true
- Safety Lock: locked_readonly（全局锁定，仅展示，不可通过 UI 解锁）
- “发送到 VS Code 查看端” = 审查查看，不是执行
- sourceMetadata / reviewMetadata / contextId / hash 不得混入 Agent 任务卡正文
- 正常模式隐藏调试字段；调试模式可显示 hash / contextId / rawReport / sourceMetadata / reviewMetadata / diagnostic fields
```

### 4.7 实现对照标准

执行端实现完毕后，如何对照基线包验证:
- 对照变更列表逐项检查
- 对照状态矩阵逐状态测试
- 对照字段层级检查正常/调试模式
- 对照 out-of-scope 确认未越界实现
- 对照安全边界声明确认未开放执行能力

---

## 5. 安全边界

以下安全边界在任何 UI 基线版本中均为强制不变，不得在 UI 设计或实现中弱化或绕过。

| 安全字段 | 值 | 含义 |
|----------|-----|------|
| `executionAllowed` | `false` | 不执行任何命令 |
| `agentDispatchAllowed` | `false` | 不派发任何 Agent |
| `gitWriteAllowed` | `false` | 不写入 Git |
| `noAutoDispatch` | `true` | 不自动派发任务卡 |
| `noCommandExecution` | `true` | 不运行任何命令 |
| Safety Lock | `locked_readonly` | 全局只读锁定 |

### 5.1 标签保护

- "发送到 VS Code 查看端" 永远不写为 "执行"、"运行"、"派发"。
- 按钮文案必须明确传达"查看/审查"，不是"执行"。
- sourceMetadata、reviewMetadata、contextId、hash 仅供 ACB 内部审查与诊断使用，不得混入 Agent 任务卡正文。
- 可复制给 Agent 的载荷只能是完整 bounded ACB_TASK_CARD，不含 ACB 内部元数据。

### 5.2 模式分离

- 正常模式: 隐藏 hash、contextId、rawReport、sourceMetadata、reviewMetadata、diagnostic fields。
- 调试模式: 上述字段可显示，但仅为开发和诊断用途。
- 模式切换不改变任何安全边界。

---

## 6. 禁止事项

### 6.1 禁止的 UI 开发行为

- ❌ 基于蓝图直接全面改写 UI（必须由总控拆分任务卡后逐步实施）
- ❌ 跳过状态矩阵直接实现交互状态变更
- ❌ 在 UI 结构版本中跳过 before-after 对比
- ❌ 在正常模式中暴露调试字段
- ❌ 在 UI 中暗示"执行"、"运行"、"派发"等目前未开放的语义
- ❌ 通过 UI 绕过安全锁或修改安全边界

### 6.2 禁止的工作流行为

- ❌ UI/UX 顾问直接向执行端派发任务卡
- ❌ 执行端根据蓝图自行判断实现范围
- ❌ 将 Figma/v0/Uizard 作为强制依赖
- ❌ 要求每个微小 bug 都产出完整蓝图
- ❌ 将静态蓝图页视为生产 UI 的直接替代品

### 6.3 不强制依赖

ACB UI 开发**不强制**使用以下工具:
- Figma
- v0
- Uizard
- React
- Tailwind
- 任何外部 CDN 或 SaaS 设计工具

静态 HTML 蓝图页 + 浏览器截图 + Excalidraw 标注是当前建议的最低成本工具链。

## 7. 工作流总览

```text
┌─────────────────────────────────────────────────────┐
│ 1. 总控定版本                                          │
│    定义 UI 变更的目标、范围、版本类型                      │
│    生成 UI 基线请求任务卡                                │
├─────────────────────────────────────────────────────┤
│ 2. UI 定界面基准                                       │
│    UI/UX 顾问产出:                                     │
│    - 目标 UI 图片 / 静态蓝图                             │
│    - 变更列表                                          │
│    - 状态矩阵                                          │
│    - 按钮文案表                                        │
│    - 字段层级                                          │
│    - out-of-scope 列表                                │
│    - 安全边界声明                                       │
│    → 交回总控审查                                       │
├─────────────────────────────────────────────────────┤
│ 3. 总控审查 + 派任务                                    │
│    总控审查基线包                                       │
│    拆分实现任务卡                                       │
│    派发给执行端                                        │
├─────────────────────────────────────────────────────┤
│ 4. 执行端落地                                          │
│    DeepSeek / Codex / Claude Code 按任务卡实现          │
│    不自行扩大范围                                       │
│    完成后回报                                          │
├─────────────────────────────────────────────────────┤
│ 5. UI 回看                                            │
│    UI/UX 顾问对照基线包审查实现结果                       │
│    记录差异和建议                                       │
│    → 交回总控                                          │
├─────────────────────────────────────────────────────┤
│ 6. 总控收口                                            │
│    验收 / 修正 / 关闭                                   │
│    更新版本记录                                        │
└─────────────────────────────────────────────────────┘
```
