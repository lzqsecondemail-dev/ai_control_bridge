# ACB 控制台静态 UI 蓝图页说明

## 用途

这是一份静态、脱敏的 HTML 设计蓝图，用于 ACB 浏览器控制台的 UI/UX 评审与设计交接。

它不连接任何真实 Bridge，不读取真实数据，不执行任何任务。

## 如何使用

### 1. 在浏览器中打开

直接用浏览器打开 `docs/03_design/acb_console_ui_blueprint.html`。

### 2. 全页截图

使用浏览器全页截图工具或扩展（如 Firefox 内置截图、Chrome DevTools 全尺寸截图），
截取完整展开的页面。

关键: 确保使用的是**全页截图**而非窗口截图，否则只能拍到浏览器可见区域。

### 3. 导入 Excalidraw

将截图导入 Excalidraw 作为底图。

### 4. 标注

在 Excalidraw 中标注:
- 保留哪些元素
- 删除哪些元素
- 移动哪些元素位置
- 折叠/展开哪些区域
- 重命名哪些标签
- 调整哪些信息层级

使用 Excalidraw 的箭头、框选、文字标注等工具。

### 5. 发送给 UI/UX 顾问

将标注后的截图发送给 UI/UX 顾问进行评审。

### 6. 发送给总控

将顾问建议发送给执行总控，由总控拆分实际实现任务。

## 这个页面是什么

- 是 UI/UX 设计交接的静态参考蓝图
- 是截图和标注的载体
- 是团队讨论 UI 结构的可视化基础
- 是对未来生产 UI 的形状参考

## 这个页面不是什么

- 不是生产 UI 实现
- 不是可交互的前端原型
- 不连接任何后端服务
- 不替代 actual browser extension UI
- 不用于实际任务执行

## 建议后续工作流

1. 在浏览器中打开 HTML
2. 全页截图
3. 导入 Excalidraw
4. 标注 keep / delete / move / collapse / rename
5. 将标注图片发送给 UI/UX 顾问
6. 将顾问建议发送给执行总控
7. 总控拆分实际实现任务卡

## 关键技术约束

- 纯静态 HTML/CSS/JS，无外部依赖
- 不使用 CDN
- 不使用框架
- 不连接 localhost
- 不调用 Chrome Extension API
- 所有数据均为脱敏假数据
