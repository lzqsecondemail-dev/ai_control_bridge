# M3-UI.2-A-1 普通模式 P0 下一版目标视觉锚点 · 说明

## 目的

创建一份静态 HTML 页面，作为 ACB 浏览器控制台**下一版本普通模式**的目标视觉锚点。

这份目标锚点回答一个问题：

> 下一版 ACB 控制台普通模式的第一屏应该长什么样，才能让用户一眼看懂当前有没有可发送任务卡，以及下一步能做什么？

它不回答：

> 所有边缘情况、调试模式、或长期蓝图应该长什么样？

---

## 为什么先做 A-0 再做 A-1

在 M3-UI.2-A-0 当前状态基线被用户接受（已基本接受，非阻塞差异可以忽略）后，方可进入下一版本目标设计。

工作流顺序：

1. **M3-UI.2-A-0** → 当前状态复刻 → 用户验证 → 被接受 ✓
2. **M3-UI.2-A-1（本页）** → 下一版目标设计 → 用户确认 → 被接受
3. **M3-UI.2-B** → 生产代码实现

跳过当前状态基线直接做目标设计，可能导致目标建立在错误假设上。

---

## A-0 验收结论

A-0 被用户评价为"basically sufficient（基本可用）"，结论如下：

- 非阻塞颜色偏差（当前反馈类型卡片颜色）可以忽略，不作为修正目标
- "检查与操作结果"面板放在任务卡下方是用户确认的目标方向
- 安全/匹配字段在普通模式中过于显眼 → 搬到"更多信息"/调试模式
- 无需修正 A-0 后再做 A-1

---

## P0 核心设计目标

> 普通用户打开控制台后，第一屏是否能立刻看懂当前有没有可发送任务卡，以及下一步能做什么？

具体落地为：

1. **任务卡区域居中放大** — 蓝色边框突出，浮动 "任务卡" 标签
2. **主操作按钮突出** — "检查可发送状态" 加大加粗，放在按钮组第一位
3. **检查结果面板在任务卡下方** — A-0 用户确认的目标位置
4. **顶部状态栏精简** — 从 9 chips 减到 6 chips，预检/就绪/审查移入"更多信息"
5. **匹配字段下移** — safety/matching 字段从显眼位置移到 "更多信息" / 调试模式
6. **右栏简化** — 工具卡片从 7 个减到 5 个，移除详细诊断面板

## 三种页面的区别

| 页面 | 文件 | 用途 |
|------|------|------|
| **长期静态蓝图** | `acb_console_ui_blueprint.html` | 长期方向参考，不代表当前状态或下一版本目标 |
| **当前状态基线** | `ACB_M3UI2A0_CURRENT_CONSOLE_STATE_BASELINE.html` | 根据生产代码复刻当前 UI（A-0，已接受） |
| **下一版本目标锚点** | `ACB_M3UI2A1_NORMAL_MODE_P0_TARGET_ANCHOR.html` | 用户确认的下一版本普通模式目标 UI，用于指导 M3-UI.2-B 实现 |

三者的关系：

```text
长期静态蓝图 (参考方向)
    ↓ (不是直接基准)
当前状态基线 (A-0)  →  用户验证  →  被接受 ✓
    ↓
下一版本目标 (A-1)  →  用户确认  →  被接受
    ↓
生产实现 (M3-UI.2-B)  →  PR  →  合并
```

## 如何使用本页

1. 在浏览器中打开 `ACB_M3UI2A1_NORMAL_MODE_P0_TARGET_ANCHOR.html`
2. 阅读 Section B 的目标控制台 mock（正常模式，紧凑）
3. 对照 Section C 的目标状态矩阵确认 6 种状态的分流正确
4. 对照 Section D 的 A-1 vs A-0 变化表确认每一项变化是否认可
5. 对照 Section E 的按钮文案表确认文案和禁用条件无误
6. 对照 Section F 的字段层级确认 Level 1/2/3 划分合理
7. 如有异议，标注具体位置和原因发回给执行端/总控
8. A-1 被接受后，方可进入 M3-UI.2-B 生产实现

## 页面结构

HTML 页面包含以下区域：

- **Section A**: 页面头部，含目标锚点/普通模式 P0/不连接 Bridge/不执行任务 标识
- **Section B**: 目标控制台 mock（顶部状态栏 → 安全锁横幅 → 三栏布局 → 状态示例条）
  - 左栏: 身份卡列表（5 张卡片 + 简单操作按钮）
  - 中栏: 反馈来源行 → 任务卡区域（蓝色边框 + 浮动标签）→ 主操作按钮行 → 检查结果面板 → Bridge 警告 → 更多信息折叠
  - 右栏: 5 个紧凑工具端点卡片
  - 状态示例条: 6 种状态卡片（未检查/可发送/警告/不可发送/Bridge未连接/无完整任务卡）
- **Section C**: 目标状态矩阵（6 行 × 6 列）
- **Section D**: A-1 vs A-0 视觉差异表（13 行）
- **Section E**: 按钮文案表（8 行，含禁止用词）
- **Section F**: 正常/调试模式字段层级（Level 1/2/3）
- **安全边界声明**
- **页脚**

## 不在范围内的内容 (Not In Scope)

- 不连接 Bridge
- 不使用 Chrome Extension API
- 不 fetch localhost
- 不读取真实数据
- 不使用外部 CDN 资源
- 不修改任何生产代码
- 不包含调试模式目标 UI
- 不包含完整的分发/执行能力
- 不替代长期静态蓝图

---

## 安全边界

本页遵守所有 ACB 安全边界：

- executionAllowed: false
- agentDispatchAllowed: false
- gitWriteAllowed: false
- noAutoDispatch: true
- noCommandExecution: true
- Safety Lock: locked_readonly
- "发送到 VS Code 查看端" = 审查查看，不是执行
- 正常模式隐藏调试字段（Level 3）
- sourceMetadata / reviewMetadata / contextId / hash 不混入 Agent 任务卡正文

---

## 与 A-0 的关系

本页基于 A-0 被接受后的反馈创建：

- A-0 提供了"当前是什么"，本页提供"下一步要改成什么"
- 本页不修正 A-0 的偏差（偏差已在 A-0 中记录为可忽略），而是直接描述目标
- 本页的设计决策（精简顶部栏、下移匹配字段、放大主按钮）均来自 A-0 验收过程中的用户反馈
