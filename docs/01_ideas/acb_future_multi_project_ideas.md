## 点子 1：多项目总控与多执行终端并行

### 场景
用户可能同时开发多个项目（例如 ACB 项目本身与一个游戏项目）。当前 ACB 以单项目 + 单总控为核心，适合个人开发者。未来可扩展为多项目并行管理。

### 核心思路
- GPT 对话侧允许多个独立总控，每个总控对应一个项目或工作区。
- 每个总控下可配置各自的测试负责人、工程参谋、安全参谋等顾问角色。
- 本地执行端可以是多套独立实例（多 PowerShell / Claude / DeepSeek / Codex），也可以是同一执行端按项目上下文切换。
- **关键约束**: 所有任务必须携带 `projectId` / `controllerId` / `workspaceId` 等项目标识，防止跨项目任务混淆。
- 各项目的本地状态（`latestFeedbacks`、`actionSteps`、`feedbackClassifications`）应相互隔离。

### 定位
适合作为个人版成熟后的团队版 / Pro 功能讨论。个人版阶段保持单总控即可。

### 状态：挂起，不进入当前开发

## 点子 2：总控界面浏览其他对话摘要

### 场景
用户停留在某个总控界面时，希望能从控制台快速看到其他对话（测试参谋、工程参谋、其他 GPT 对话）的简要摘要，而不必切换到对应页面。

### 核心思路
- 在浮动控制台中提供一个"跨对话摘要"面板，列出其他通道的最新反馈摘要（标题 + 一句话摘要 + 状态）。
- 初期只读浏览，不允许跨页面修改状态或回复。
- 后续可讨论是否允许在摘要面板中直接进行标记（标记已读/挂起）、转交任务给当前总控、或一键跳转到对应对话页面。
- 这属于"多通道总览 / 跨对话摘要面板"方向，不进入当前 M2-B.1-A。

### 状态：挂起，不进入当前开发

## 点子 3：VS Code 扩展状态面板深化与产品化

### 背景
M3-A.0 VS Code Extension Project Status Foundation 已完成（commit `9d80810`）。当前扩展提供了：
- `ACB: Show Project Status` Webview Panel
- `ACB: Copy Project Status Summary` 剪贴板命令
- Git API + CLI 回退双路径
- Runtime Diagnostics（26 个运行时字段）

扩展本身的定位已从"独立状态面板"明确为"ACB 本地执行桥的基础模块"。

### 已暂缓的具体方向

以下方向均为 M3-A.0 完成后识别出的后续扩展可能，已暂缓：

| 项目 | 暂缓原因 |
|------|----------|
| M3-A.1 状态显示扩展 | 无 Bridge 时继续深化 UI 回报递减 |
| Runtime Diagnostics 产品化 | 当前为开发者调试阶段，后续应折叠或仅在 dev 模式下暴露 |
| `ACB: Show Project Status` 作为主用户流 | 后续应降级为 status bar / 项目状态卡片 / 预检面板 |
| Copy Summary 独立命令 | 后续可融入自动执行报告生成 |
| 多 workspace 支持 | 当前阶段单 workspace 够用 |
| 状态自动刷新 | 后续可在 Bridge 连接后按需触发 |

### 未来调用时机（Bridge 连通后）

VS Code 扩展现有能力将在以下 Bridge / 执行流程中复用：

a. 执行前项目匹配 — 确认 Bridge 连接的 workspace 与任务卡 `projectDir` 一致
b. branch/commit 预检 — 执行前验证当前分支和提交是否与任务卡匹配
c. dirty workspace 警告 — 执行前发现未提交变更时发出警告
d. allowedFiles/forbiddenActions 变更对比 — 执行后检查是否超出限定范围
e. 执行结果收集 — 任务完成后收集 Git 状态作为执行回报的一部分
f. 本地执行报告生成 — 将项目状态、变更摘要、执行结果打包为回报
g. Bridge 本地端点 — 作为 browser-to-local Bridge 消息的本地接收端

### 开发备忘

- `acb-status-test` 是开发测试仓库，非产品依赖
- launch.json folder-uri 格式需使用 `file:///D%3A/...` 标准编码，不可用裸 Windows 路径
- Runtime Diagnostics 中的 26 个字段为开发者阶段工具，产品化时应裁剪

### 状态：挂起。所有项目在 Bridge 最小通信回路建立后重新评估优先级。
